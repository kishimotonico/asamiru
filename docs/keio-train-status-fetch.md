# 京王線・京王相模原線の列車到着見込み取得方法

## 目的

現在日時を基準に、任意の駅から京王線・京王相模原線に乗る場合に、次の電車について以下を取得する。

- 行先
- 種別
- 定刻発車時刻
- 遅延分数
- 遅延込みの見込み発車時刻

例:

```txt
橋本行き / 各停 / 調布駅 08:12発 / 3分遅れ / 見込み 08:15発
```

## 使用するエンドポイント

### 列車走行位置・遅延

```http
GET https://i.opentidkeio.jp/data/traffic_info.json
```

認証なしで取得できる。レスポンスには、現在走行中または駅停車中の列車情報が含まれる。

主なレスポンス構造:

```json
{
  "up": [
    {
      "dt": [
        {
          "yy": "2026",
          "mt": "05",
          "dy": "30",
          "hh": "07",
          "mm": "27",
          "ss": "30"
        }
      ],
      "st": "0"
    }
  ],
  "TS": [],
  "TB": []
}
```

主な項目:

- `up`: データ更新時刻
- `TS`: 駅停車中の列車一覧
- `TB`: 駅間走行中の列車一覧
- `TS[].id` / `TB[].id`: 駅または駅間を表す ID
- `TS[].sn` / `TB[].sn`: 路線・系統を表す値
- `TS[].ps` / `TB[].ps`: その駅・駅間にいる列車一覧

列車データの主な項目:

- `tr`: 列車番号。列車別時刻表エンドポイントの `{trainId}` に使う
- `sy`: 種別コード
- `sy_tr`: 実際の種別コード
- `ki`: 方向
- `bs`: 番線・線路
- `dl`: 遅延分数
- `ik`: 行先コード
- `ik_tr`: 実際の行先コード
- `sr`: 両数
- `inf`: 種別変更などの案内文

### 列車別時刻表

```http
GET https://i.opentidkeio.jp/dia/{trainId}.json
```

`{trainId}` には、列車走行位置・遅延 API の `tr` を前後空白を除去して指定する。

例:

```http
GET https://i.opentidkeio.jp/dia/0105.json
```

レスポンス例:

```json
{
  "dy": [
    {
      "st": "1",
      "sn": "新宿",
      "tt": "",
      "ht": "7:28",
      "pa": "1"
    },
    {
      "st": "2",
      "sn": "笹塚",
      "tt": "7:32",
      "ht": "7:34",
      "pa": "1"
    }
  ]
}
```

主な項目:

- `dy`: 駅ごとの時刻表
- `st`: 駅順 ID
- `sn`: 駅名
- `tt`: 到着時刻と思われる値
- `ht`: 発車時刻と思われる値
- `pa`: 停車有無のフラグと思われる値

乗車駅の発車時刻を出す用途では、基本的に `ht` を使う。`ht` が空の場合は、その駅に停車しない、または発車時刻がない列車として扱う。

## 種別・行先・遅延の扱い

列車走行位置・遅延 API の列車データから取得する。

- 列車番号: `tr`
- 遅延分数: `dl`
- 種別: 原則として `sy_tr` を使い、必要に応じて `sy` を参照する
- 行先: 原則として `ik_tr` を使い、必要に応じて `ik` を参照する

種別コードの対応例:

- `1`: 特急
- `2`: 急行
- `3`: 快速
- `4`: 準特急
- `5`: 区間急行
- `6`: 各駅停車
- `9`: 京王ライナー
- `10`: 臨時
- `11`: Mt.TAKAO号

## 次の電車を求める流れ

### 1. 列車走行位置・遅延 API を取得する

`https://i.opentidkeio.jp/data/traffic_info.json` を取得し、`TS` と `TB` に含まれる列車を候補にする。

### 2. 対象外の列車を除外する

京王本線・京王相模原線だけを対象にする場合、以下を除外する。

- 井の頭線など、対象外系統の列車
- 乗車対象にしない支線・区間の列車
- すでに乗車駅を通過している列車

「すでに乗車駅を通過しているか」を判定するには、列車の現在位置、進行方向、乗車駅の駅順を比較する。

### 3. 列車ごとの時刻表を取得する

各候補列車について、列車番号 `tr` を使って列車別時刻表を取得する。

```http
GET https://i.opentidkeio.jp/dia/{trainId}.json
```

### 4. 乗車駅の発車時刻を取得する

列車別時刻表の `dy` から、乗車駅名に一致する `sn` の行を探す。

- `sn` が乗車駅名と一致する行を使う
- その行の `ht` を定刻発車時刻として扱う
- `ht` が空なら、その列車は乗車駅に停車しない候補として除外する

### 5. 遅延込みの見込み発車時刻を計算する

列車走行位置・遅延 API の `dl` を、列車別時刻表の `ht` に加算する。

例:

```txt
定刻発車時刻: 08:12
遅延分数: 3
見込み発車時刻: 08:15
```

### 6. 現在時刻以降の最短候補を選ぶ

見込み発車時刻が現在時刻以降の列車だけを残し、その中で最も早い列車を「次の電車」とする。

最終的な出力項目は以下を想定する。

- 乗車駅名
- 列車番号
- 行先
- 種別
- 定刻発車時刻
- 遅延分数
- 見込み発車時刻

## 注意点

- この API は ODPT ではない。
- 公開仕様として安定保証されている API ではなさそうなので、フィールド変更やレスポンス欠落に備える必要がある。
- `traffic_info.json` は頻繁に更新されるため、短時間に過剰なリクエストを送らない。目安として 5 秒以上の間隔を空ける。
- `dia/{trainId}.json` は列車ごとに取得が必要になるため、候補列車を絞ってから取得する。
- 厳密な「次の電車」判定には、列車の現在位置・進行方向・乗車駅の駅順を使って、乗車駅を通過済みの列車を除外する必要がある。
- まずは「乗車駅の `ht` + `dl` が現在時刻以降で最も早い列車」を返す方式で、実用に近い到着見込みを出せる。

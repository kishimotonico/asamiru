# 路線運行情報の誤表示修正

## 問題

設定上の路線名と Yahoo URL がずれた場合に、別路線の運行情報が誤った路線名で表示されるケースがあった。

## 原因

運行情報 API がクライアントから渡された路線名をそのまま表示名に使っていたため、保存済み設定やカスタム設定で `name` と `yahooUrl` がずれると、別路線の情報を指定路線名で表示できた。

また、Yahoo!路線情報のパースで本文全体からステータスを推測するフォールバックがあり、関連つぶやきなど運行情報本体ではないテキストを拾う余地があった。

## 対応

- Yahoo! Transit URL を `https://transit.yahoo.co.jp/diainfo/{rail}/{range}` に限定・正規化。
- 表示名はクライアント入力ではなく、取得した Yahoo ページの見出しから採用。
- パース対象を `#mdServiceStatus` の運行情報ブロックに限定。
- 本文全体からのステータス推測フォールバックを削除し、必要な要素が読めない場合は失敗させる。
- API キャッシュ削除エンドポイントを追加。
- Web ビルドを通すため、運行情報表示キーと Jotai storage 型を修正。

## 確認

- `pnpm build`
- `POST /api/train-status` に誤った路線名と別路線の URL を送っても、API は取得元ページの路線名として返すことを確認。

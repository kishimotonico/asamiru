## 把握した非直感ポイント（要点）

- 起床帯のど真ん中で手動スリープ（`s`）すると、`clearManual` が「起床帯への流入エッジ」でしか発火しないため、操作するまでその起床帯の残り時間ずっとスリープのまま。
- 外部 OFF の確定に最大10秒（OFF 連続2回 × POLL 5秒）。
- `putDesiredPower` 失敗はサイレント（warn のみ）。アプリは黒画面だが物理モニターは点いたまま。
- `useDisplaySync` の init は1回きりで、失敗すると `enabled=false` のまま再試行しない（起動順序・瞬断に弱い）。

## 改善提案（詳細は仕様書）

- A. モニター連動を SSE 常時接続化して初期化リトライを得る（破壊度小・最初に着手推奨）。
- B. 表示判定を `desiredSleeping` 一本へ正規化し `effectiveSleeping` の合成点を消す（中）。
- C. 起床帯中の手動スリープをワンショット化して最大の非直感を解消（中）。
- D. フラグ（`manualSleeping` ＋ `awakeUntil`）をやめ、明示的な `SleepIntent` 判別共用体（`schedule` / `tempAwake` / `forcedSleep`）へ統一（大・本命。B・C を内包）。
- E. サーバー権威化は要件が出るまで保留（ADR で棄却済み）。

実装するなら D を軸に B・C を内包、前段に A を独立で。
